Fix for "auth/operation-not-allowed" Error

You're getting the auth/operation-not-allowed error when trying to create an account because Email/Password Authentication is not enabled in your Firebase project.

Note: Even though the Google authentication works, Email/Password authentication needs to be separately enabled.

Solution: Enable Email/Password Authentication

Step 1: Open Firebase Console

Go to Firebase Console and select your project: ymovies-e4cb4

Step 2: Navigate to Authentication

In the left sidebar, click on Authentication

Step 3: Go to Sign-in Methods

Click on the Sign-in method tab at the top

Step 4: Enable Email/Password

Find Email/Password in the list of providers

Click on it to expand

Toggle the Enable switch to ON position

Click Save

Step 5: Test Again

Return to your application and try creating an account again

The signup should now work properly

Why This Error Occurs

The auth/operation-not-allowed error is thrown when you try to use an authentication method that isn't enabled in the Firebase Console. It's a security measure to prevent unintended authentication methods from being used in your application.

Additional Troubleshooting

  • Make sure your Firebase configuration in .env.local and client/public/env.js is correct
  • Check that you're using the correct Firebase project (ymovies-e4cb4)
  • If issues persist, check the browser console for more detailed error messages

Once you enable Email/Password authentication, the issue with creating accounts via email/password will be resolved, and users will be able to create accounts without requiring Google authentication.
